# Мой репозиторий devops-netology

## .gitignore правила

В этом репозитории используются следующие правила игнорирования:

- В корне: общий `.gitignore` (пока пуст, но зарезервирован для будущих глобальных исключений).
- В каталоге `terraform/`: применяется специфичный `.gitignore` для Terraform, который исключает:
  - Файлы состояния (`*.tfstate`, `*.tfstate.*`)
  - Локальные переменные (`*.tfvars`)
  - Каталог `.terraform/` с загруженными провайдерами
  - Планы выполнения (`*.tfplan`)
  - Файлы журналов и резервные копии

Это предотвращает попадание чувствительных данных и артефактов в систему контроля версий.
